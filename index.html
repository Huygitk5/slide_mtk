1.1. Khái niệm & Bối cảnh

Bản chất

  • Thuộc nhóm Structural (Cấu trúc) của Gang of Four.
  • Tổ chức, kết hợp các lớp và đối tượng lại với nhau.

Mục đích cốt lõi

  • Đính kèm tính năng/trạng thái mới ĐỘNG tại thời điểm chạy (Runtime).
  • Không ảnh hưởng đến đối tượng khác.
  • Không thay đổi mã nguồn lớp ban đầu.

1.2. Vấn đề: Sự bùng nổ lớp (Class Explosion)

Cách tiếp cận cũ: Kế thừa (Inheritance)

Áp dụng tĩnh ở cấp độ lớp. Để có Cà phê + Sữa + Caramel, ta phải tạo class CaramelMilkCoffee. Càng nhiều Topping, số lượng class tăng theo cấp số nhân!

Giải pháp của Decorator: Bao gộp (Composition)

Tạo ra các lớp "vỏ bọc" độc lập. Mỗi vỏ đại diện cho 1 Topping. Cần gì thì bọc nấy ngay lúc chạy (Runtime) ➔ Dập tắt sự bùng nổ lớp.

1.3. Cấu trúc 4 thành phần cốt lõi

Thành phần Bản chất Trách nhiệm
1. Component Interface / Abstract Giao diện chung. Giúp Client không phân biệt được Lõi hay Vỏ bọc.
2. Concrete Component Class (Lõi) Xác định hành vi nền tảng nhất trước khi bị trang trí.
3. Base Decorator Abstract Class Giữ tham chiếu (Pointer) đến Component. Ủy quyền (Delegate) công việc cho đối tượng bên trong.
4. Concrete Decorators Class (Kế thừa) Chứa logic mới. Gọi hàm lớp cha + Thực hiện thêm logic riêng.

1.4. Cơ chế hoạt động (Củ hành tây)

Hoạt động như một ngăn xếp (Stack) đệ quy ngược.

Luồng thực thi:

  1. Client gọi hàm từ lớp ngoài cùng (C).
  2. C xử lý, ủy quyền gọi hàm cho B.
  3. B xử lý, ủy quyền gọi hàm cho A (Lõi).
  4. A hoàn thành nền tảng, trả kết quả về B.
  5. B nhận, thêm logic, trả về C.
  6. C nhận, thêm logic cuối, trả về Client.
Vỏ C (Decorator)
Vỏ B (Decorator)
Lõi A

1.5. Đánh giá chuyên sâu

Ưu điểm (Chuẩn SOLID)

  • Linh hoạt: Thêm/bớt tính năng lúc Runtime.
  • SRP (Đơn trách nhiệm): Chia class khổng lồ thành nhiều class nhỏ.
  • OCP (Đóng/Mở): Viết thêm Decorator mới mà không sợ hỏng code cũ.

Nhược điểm

  • Khởi tạo dài dòng: Cần lồng nhiều `new()`. Thường phải dùng kèm Builder/Factory Pattern.
  • Khó gỡ lỗi (Debug): Trace code vất vả do nhảy qua lại nhiều lớp.
  • Phụ thuộc thứ tự bọc: Bọc sai thứ tự có thể ra kết quả sai logic nghiệp vụ.

2.3. Code Bước 1: Xây dựng nền móng (Interface & Core)

1. Component & Concrete Component 
// Giao diện chung bắt buộc mọi Lõi và Vỏ bọc phải tuân thủ
public interface ITableBooking
{
    string GetBookingInfo();
    decimal GetDepositFee();
}

// Lõi cơ bản (Bàn tiêu chuẩn)
public class BasicBooking : ITableBooking
{
    public string GetBookingInfo() 
        => "Bàn tiêu chuẩn";

    public decimal GetDepositFee() 
        => 50000; // Cọc cơ bản
}

2.4. Code Bước 2: Lớp vỏ bọc cơ sở (Base Decorator)

2. Base Decorator (Trái tim của Pattern) 
public abstract class BookingDecorator : ITableBooking
{
    // Biến tham chiếu trỏ tới lớp Lõi hoặc Vỏ bọc nằm bên trong nó
    protected ITableBooking _booking; 

    public BookingDecorator(ITableBooking booking)
    {
        _booking = booking;
    }

    // Cơ chế ỦY QUYỀN (Delegate) thực thi
    public virtual string GetBookingInfo() 
        => _booking.GetBookingInfo();

    public virtual decimal GetDepositFee() 
        => _booking.GetDepositFee();
}

2.5. Code Bước 3: Các lớp bọc chức năng cụ thể

Bọc tính năng cơ bản
// Vỏ bọc: Bàn gần cửa sổ (+20k)
public class WindowSeatDecorator : BookingDecorator
{
    public WindowSeatDecorator(ITableBooking b) 
        : base(b) { }

    public override string GetBookingInfo()
    {
        // Lấy Info cũ + Nối thêm thông tin mới
        return base.GetBookingInfo() + " + Cửa sổ";
    }

    public override decimal GetDepositFee()
    {
        // Lấy tiền cũ + Cộng thêm 20k
        return base.GetDepositFee() + 20000;
    }
}
Bọc tính năng có tham số
// Vỏ bọc: Ghế trẻ em (Truyền số lượng)
public class BabyChairDecorator : BookingDecorator
{
    private int _qty; // Số lượng

    public BabyChairDecorator(ITableBooking b, int qty) 
        : base(b)
    {
        _qty = qty;
    }

    public override decimal GetDepositFee()
    {
        // Tiền cũ + (50k * Số lượng)
        return base.GetDepositFee() + (50000 * _qty);
    }
    /* GetBookingInfo() tương tự... */
}
